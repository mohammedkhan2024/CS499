Software Design and Engineering Enhancements

Artifact Name: Family Expense Tracker App

What the Artifact Is and Its Purpose

The Family Expense Tracker App is a budgeting and expense management web app built using Python and Streamlit. It is designed to help users add family members, record expenses, track spending patterns, and view financial summaries.

Why It Needed Enhancement

When I first found the Family Expense Tracker app, it was built as a single long Python file. The logic, user interface, data management, and layout all existed together in one long script. Due to the fact that the entire app was written in one script, it made it hard to manage, hard to test, and nearly impossible to scale or improve without breaking something. Features like error handling, form validation, and session tracking were completely missing. The app also used the default Streamlit styling, which looked plain and did not provide a helpful or guided experience for users.

There were no charts, no weekly or daily tracking, and no sense of structure in the codebase, basically everything lived in one place. The app had no logging system to track actions or issues, and no testing setup to catch bugs early. It lacked any method of deployment and would only run on the developer’s machine after manually installing everything. There were also no motivational elements for the user, basically nothing like gamification or reward systems that would encourage better budgeting habits. In short, while the basic idea was there, the app needed significant engineering improvements to become a usable and shareable product.

Why I Included This Artifact in My ePortfolio

I selected this project for my ePortfolio because it demonstrates my ability to thoughtfully apply software engineering practices across the full development lifecycle. It showcases how I took a simple working idea and expanded it using both my technical skills and direct suggestions from the enhancement guidelines provided by my professor. For example, I implemented Docker containerization to meet the recommendation and added gamification features like a reward badge system to recognize consistent budget behavior.

The project also reflects how I used design principles from the redesign using UI/UX principles suggestion. I improved the app's layout using st.columns(), created custom visual styling through a CSS file and used clear icons and headings to improve the user experience. These enhancements are more than visual polish but they reflect thought given to accessibility and user clarity.

The biggest difference from the original version is the structure and quality of the code. I refactored everything into a modular design, separating business logic, validation, logging, and UI components into their own folders. I added a full input validation layer and wrapped key logic in error handling blocks to prevent crashes. I also created a tests/ folder and added tests to verify my core functions. Logging was introduced to track actions like added members or failed submissions, which is something I never had in the original version. Finally, I created a Dockerfile to allow anyone to run the app easily, which is a big improvement from needing to manually install packages and troubleshoot dependencies.

This artifact grew well beyond its original scope which is why it stands out as the best item to show what I can do in software design. It doesn't just show that I can build an app but it shows I can enhance and engineer a product with professionalism and purpose.

Did I Meet My Course Outcomes?

Yes, I met all the course outcomes I planned to address at the beginning of Module One. I designed my application to be organized, modular, reusable, and deployable. I added strong validation and error handling to make it more reliable. I improved the user experience using thoughtful UI/UX improvements. I wrote and executed unit tests, and I deployed the app using Docker for professional level distribution.

In addition to Docker, I used Streamlit Cloud to professionally host the app online. This allowed me to make the project publicly accessible and easy to share with a broader audience, which include potential employers. Hosting the app through a cloud platform helped me understand not only how to build an app but also how to deploy it in a real world setting.

I also expanded my original plan by including new elements like gamification. These additions strengthened the outcome coverage and added new areas like user motivation.

What I Learned from This Project

This project helped me better understand what goes into professional software design. At first, the app was a simple script, but as I broke the code into modules, I saw how clean architecture leads to better flexibility and maintainability. I learned that organizing code isn’t just about structure. it helps the app grow naturally which makes enhancements easier.

I also learned that good design isn’t just visual, it's how you guide and support the user. The UI/UX improvements helped make the app clearer and more enjoyable. Adding gamification shows how rewarding feedback encourages behavior of the user and makes an app feel alive. Docker helped me see the importance of packaging software properly. Now I can share my work easily and know it will run the same anywhere which is a real world skill that matters.

Algorithms and Data Structure Enhancements

Artifact Name: Family Expense Tracker App

What the Artifact Is and Its Purpose

The artifact is the Family Expense Tracker, an app built with Python and Streamlit that helps people manage their family budget. It lets users add family members, track income and expenses, set budgets, and see where their money is going over weeks or months.

Why It Was Selected

I picked this project to improve because it uses important computer science concepts like data structures and algorithms. I wanted to make sure the app handles data correctly and safely. Adding features like undo and redo helps prevent mistakes and makes the app more reliable for users.

How the Artifact Was Enhanced

To implement algorithms and data structure features, I improved the app by adding better filtering so users can choose which expenses to see based on date, category, and amount. I also added sorting so they can organize expenses by date, amount, or category. Users can now manually pick any week or month to check their budget and spending.

To help users manage their money better, I created budget setters for both weekly and monthly budgets. Users can set their own limits, and the app tracks how much they’ve spent in selected periods. It shows how much is left and uses progress bars to visually indicate if they are staying within their budget or going over.

To make the app faster, I added a special data structure called a max-heap that quickly finds the biggest expenses. I also built an undo/redo system that remembers actions so users can easily fix mistakes. I tested these new features with different cases to make sure they work well.

Connection to Course Objectives

These improvements show that I can design smart algorithms and use data structures like heaps and stacks in real projects. I also practiced writing tests to catch problems early. The work supports the course goals of building efficiently and especially a user friendly software.

Personal Reflection on the Learning Experience

Working on these enhancements helped me understand how choosing the right data structures makes a big difference in app performance. Building undo/redo taught me how to make software more user friendly and safe. The dynamic date filtering showed me how to make apps flexible to what users need. Creating the weekly and monthly budget trackers showed me how to provide clear and helpful feedback to users about their spending habits.

Databases Enhancements

1. What database parts did I work on, and how do they make the appication better?

I worked on integrating the SQLite database into the app to replace its temporary data storage for family members and expenses. This involved designing the database structure, writing code to perform CRUD operations, and updating the app's tracking and user interface components to use this new database. This significantly improves the project by providing persistent data storage and making sure user data is saved between sessions which enhanced overall reliability.

2. What specific skills did I show when working on the database?

Through my contributions to database enhancement, I demonstrated a range of specific skills. These include designing the database schema along with structuring how data is stored efficiently and writing SQL queries to interact with and manipulate the database. I also showed my ability to integrate a relational database with a Python application and refactor existing code to work seamlessly with persistent storage. Furthermore, I proved proficient in managing data consistency and converting between database records and application objects efficiently. I also practiced maintaining code modularity and applying best practices for database operations within a web application context.

3. Any problems I faced during this part of the project, and how did I fix them?

One important challenge I encountered was making sure there is a seamless conversion between database rows and model objects to maintain consistent application logic. I resolved this by implementing factory methods in my model classes which allowed easy creation of instances from database tuples. Another challenge involved managing cached data in the UI to make sure the display always showed fresh information. Thorough testing and debugging were also crucial in fixing issues related to date conversion and data persistence.

4. How did I make sure the app was secure and performed well?

To enhance security, I used parameterized SQL queries, which are important to prevent vulnerabilities like SQL injection attacks and enabled foreign key constraints in SQLite to maintain data integrity. For performance, I optimized queries by limiting data retrieval to necessary fields and used efficient data structures like heaps to quickly retrieve top expenses.

5. How did I make the app easier and better for users in this part of the project?

By integrating the database, I significantly improved the user experience by allowing data persistence so users no longer lose information upon app restarts. The interface now consistently reflects up to date and accurate information pulled directly from persistent storage. Features such as undo/redo actions and filtering, sorting, and visualization capabilities now operate reliably on a consistent dataset which provides a smoother and more dependable, and professional user experience.
